Asset Monitoring - Available Commands
======================================

Setup:
  rake install / make install       - Install dependencies via Bundler

Development:
  rake server / make server         - Start the application server (default 0.0.0.0:8080)
  rake dev / make dev               - Start with auto-reload (requires rerun gem)
  rake console / make console       - Start an interactive Pry console

Testing:
  rake spec / make spec             - Run RSpec tests
  rake test / make test             - Alias for spec
  rake rubocop / make rubocop       - Run RuboCop linter
  rake lint / make lint             - Alias for rubocop
  rake check / make check           - Run all checks (rubocop + specs)
  rake coverage / make coverage     - Run tests with coverage report

Security:
  rake security / make security     - Run bundle-audit and brakeman
  rake audit / make audit           - Run bundle-audit only
  rake brakeman / make brakeman     - Run brakeman only

Podman:
  rake podman:build / make podman-build   - Build image as quay.io/dkirwan/asset-monitoring:dev
  rake podman:run / make podman-run       - Run quay.io/dkirwan/asset-monitoring:latest (365-day retention, SQLite in ./data)

  Image tags (override via env):
    BUILD_IMAGE=quay.io/dkirwan/asset-monitoring:dev   - Tag for podman:build
    IMAGE=quay.io/dkirwan/asset-monitoring:latest      - Tag for podman:run
    USE_DEV_IMAGE=true                                 - Run the :dev image instead of :latest

  podman:run defaults (override via env):
    PRICE_HISTORY_RETENTION_DAYS=365      - Retain price history for one year
    PRICE_HISTORY_DB_PATH=/data/asset_history.db  - Path inside the container
    PORTFOLIO_DB_PATH=/data/portfolio.db          - Portfolio holdings + history (defaults to same dir as price history)
    DATA_DIR=./data                       - Host directory bind-mounted to /data

  The SQLite files appear in ./data/ after a successful start
  (asset_history.db for spot prices, portfolio.db for holdings and portfolio history).
  Check container logs for "SQLite price history enabled" or permission errors.

Environment overrides (server/dev):
  HOST=0.0.0.0 PORT=8080            - Bind address and port (defaults shown)

Web UI (open in a browser, server on 0.0.0.0:8080):
  /                  - Redirects to /portfolio
  /portfolio         - Track holdings (gold, silver, BTC, ETH, stocks, cash, property) with history charts
  /dashboard         - Price charts (BullionVault + Coinbase; one point per background scrape)

API / metrics (curl or similar):
  /api/price_history.json    - JSON time series for the dashboard (same underlying data as /metrics, parsed). Includes retention_days.
  /api/portfolio_history.json - Per-asset and total portfolio values in EUR/USD/GBP/JPY over time (snapshots on save and scrape)
  /api/spot_prices.json      - Latest spot prices for portfolio valuation
  /api/portfolio.json        - Load (GET) and save (PUT/POST) portfolio holdings
  /metrics                - Prometheus text exposition (cached from last successful scrape)
  /health                 - Liveness JSON probe
  /ready                  - Readiness JSON probe

Optional SQLite persistence (survives restarts):
  Set PRICE_HISTORY_DB_PATH and (optionally) PRICE_HISTORY_RETENTION_DAYS.
  See README.asciidoc for Kubernetes PVC + replica considerations.
